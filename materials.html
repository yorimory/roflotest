1. Алгоритмы и блок-схемы

Что такое алгоритм?

Алгоритм — это точное и понятное предписание исполнителю совершить определенную последовательность действий для решения задачи за конечное время.

Свойства алгоритма:
  • Дискретность: разбиение на отдельные шаги.
  • Детерминированность (точность): каждое действие строго определено, нет места двусмысленности.
  • Конечность (результативность): алгоритм должен завершиться и выдать результат за конечное число шагов.
  • Массовость: алгоритм должен решать целый класс подобных задач (для разных входных данных).
  • Понятность: команды должны входить в систему команд исполнителя.

Блок-схемы (ГОСТ)

Визуальное представление алгоритма с помощью стандартизированных геометрических фигур:

ФигураНазначение
Овал (эллипс)Начало и Конец алгоритма.
ПараллелограммВвод и Вывод данных.
ПрямоугольникДействие (вычисление, присваивание).
РомбПроверка условия (ветвление: ветки Да / Нет).
ШестиугольникЦикл со счетчиком (параметрический цикл).

Базовые структуры алгоритмов

  • Следование (линейный): команды выполняются строго друг за другом.
  • Ветвление: выбор одного из путей в зависимости от условия (полное и неполное).
  • Цикл: многократное повторение действий (с предусловием, с постусловием, параметрический).
2. Системы счисления (Триады и Тетрады)

Основы систем счисления

Система счисления — способ записи чисел с помощью символов (цифр). Основание системы — это количество используемых цифр.

  • Двоичная (bin): основание 2 (цифры 0, 1).
  • Восьмеричная (oct): основание 8 (цифры 0-7).
  • Шестнадцатеричная (hex): основание 16 (цифры 0-9 и A-F, где A=10, F=15).
Перевод в десятичную: Каждая цифра умножается на основание системы в степени её разряда (начиная с 0 справа).
Пример: 1012 = 1×22 + 0×21 + 1×20 = 4 + 0 + 1 = 510

Триады и Тетрады

Для быстрого перевода между системами 2, 8 и 16 используются группы битов.

  • Триада (3 бита) — используется для связи Двоичной и Восьмеричной систем. 1 восьмеричная цифра = 1 двоичная триада.
    Пример: 78 = 1112, 58 = 1012.
  • Тетрада (4 бита) — используется для связи Двоичной и Шестнадцатеричной систем. 1 hex цифра = 1 двоичная тетрада.
    Пример: F16 = 11112, A16 = 10102.

Правило быстрого перевода: разбиваем двоичное число на группы по 3 (или 4) бита справа налево и заменяем каждую группу на соответствующую цифру.

3. Компьютерная арифметика (Коды чисел)

Чтобы процессор мог складывать и вычитать числа с помощью одних и тех же электронных схем (сумматоров), вычитание заменяется сложением с отрицательным числом. Для представления отрицательных чисел используются специальные коды.

1. Прямой код

Используется для хранения положительных чисел. Старший бит (самый левый) — знаковый. 0 = плюс, 1 = минус.

  • Прямой код числа +5 (в 8-битном формате): 00000101
  • Прямой код числа -5: 10000101

2. Обратный код

Используется как промежуточный этап для отрицательных чисел.

  • Для положительных чисел обратный код равен прямому.
  • Для отрицательных: знаковый бит остается 1, а все остальные биты инвертируются (0 на 1, 1 на 0).
  • Обратный код -5: 11111010

3. Дополнительный код (Самый важный!)

Именно в этом коде современные компьютеры хранят отрицательные числа и выполняют вычитание.

  • Для положительных чисел дополнительный код равен прямому.
  • Для отрицательных: Обратный код + 1 к младшему разряду.
  • Дополнительный код -5: Обратный (11111010) + 1 = 11111011.

Зачем? При сложении числа в прямом коде с его же дополнительным кодом (5 + (-5)) все разряды обнуляются, и мы получаем ровно 0 без усложнения процессора.

4. Динамические структуры данных

В отличие от статических массивов, динамические структуры могут изменять свой размер во время выполнения программы, выделяя и освобождая оперативную память.

Списки (Связные списки)

  • Односвязный список: каждый узел хранит данные и указатель на следующий узел.
  • Двусвязный список: каждый узел хранит указатели на следующий и предыдущий узлы (позволяет двигаться в обе стороны).

Стек (Stack)

Работает по принципу LIFO (Last In, First Out) — «последним пришел, первым ушел».

Аналогия: стопка тарелок или патроны в магазине автомата.

Применение: стек вызовов функций (рекурсия), отмена действий (Ctrl+Z).

Очередь (Queue)

Работает по принципу FIFO (First In, First Out) — «первым пришел, первым ушел».

Аналогия: очередь в магазине (кто первый встал, того первого обслужили).

Применение: буфер обмена, диспетчеризация задач в процессоре.

Деревья

Иерархическая структура. Верхний узел — корень, узлы без потомков — листья. В бинарном дереве каждый узел имеет не более двух потомков.

5. Структурное программирование и ООП

Структурный подход

Основан на отказе от безусловных переходов (goto) и использовании подпрограмм (функций) и трех базовых конструкций (следование, ветвление, цикл).

Циклы в C++

  • while (предусловие): проверяет условие ДО тела цикла. Может не выполниться ни разу.
  • do-while (постусловие): проверяет условие ПОСЛЕ тела. Гарантированно выполнится минимум 1 раз.
  • for (со счетчиком): удобен, когда количество итераций известно заранее.

Контейнеры STL (C++)

КонтейнерОписание
