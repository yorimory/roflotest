📄 Алгоритмы и способы их описания

Понятие алгоритма

Алгоритм — это точное предписание исполнителю выполнить определенную последовательность действий для достижения цели за конечное время.

Ключевые свойства алгоритма:

  • Дискретность: процесс разбивается на отдельные, завершенные шаги.
  • Детерминированность (Точность): каждое действие однозначно, нет места для свободного толкования.
  • Результативность (Конечность): алгоритм должен завершиться и выдать результат.
  • Массовость: алгоритм решает целый класс задач с разными исходными данными.
  • Понятность: команды должны быть известны исполнителю.

Способы описания:

Словесный (текст), Псевдокод (упрощенный язык программирования), Блок-схемы (ГОСТ) — графическое представление, где блоки соединены стрелками.

  • Овал — Начало / Конец.
  • Параллелограмм — Ввод / Вывод.
  • Прямоугольник — Действие / Вычисление.
  • Ромб — Условие / Ветвление.
📄 Примеры алгоритмов и базовые структуры

Три базовые структуры алгоритмов

Любая программа может быть написана с использованием только трех конструкций:

  • Следование (Линейный): блоки команд выполняются строго друг за другом, сверху вниз.
  • Ветвление (Разветвляющийся): в зависимости от условия выполняется либо одна, либо другая ветка. Бывает полным (есть действия и для ДА, и для НЕТ) и неполным (действия только для ДА).
  • Цикл (Циклический): многократное повторение участка кода.

Виды циклов:

  1. Цикл с предусловием (while): сначала проверяется условие, затем выполняется тело. Может не выполниться ни разу.
  2. Цикл с постусловием (do-while): сначала выполняется тело, затем проверяется условие. Выполнится минимум 1 раз.
  3. Цикл со счетчиком (for / параметрический): количество повторений известно заранее.
Классические примеры: Поиск максимума в массиве (цикл + условие внутри), Алгоритм Евклида (нахождение НОД вычитанием или делением), Сортировка пузырьком.
📄 Системы счисления

Основные понятия

Система счисления (СС) — это знаковая система, в которой числа записываются по определенным правилам с помощью цифр. Основание СС — количество используемых цифр.

  • Двоичная (Binary): основание 2 (цифры 0, 1). Основа работы компьютеров.
  • Восьмеричная (Octal): основание 8 (цифры 0-7).
  • Шестнадцатеричная (Hexadecimal): основание 16 (цифры 0-9 и A, B, C, D, E, F). Применяется для компактной записи двоичных данных (например, MAC-адреса, цвета HTML).

Переводы между системами

В десятичную: Каждая цифра умножается на основание системы в степени разряда (справа налево, начиная с 0).
Пример: 101₂ = 1×2² + 0×2¹ + 1×2⁰ = 4 + 0 + 1 = 5₁₀

Из десятичной: Последовательное деление числа на основание новой системы. Остатки от деления, записанные в обратном порядке, образуют новое число.

📄 Триады и Тетрады

Быстрый перевод без десятичной системы

Поскольку 8 и 16 являются степенями двойки (8 = 2³, 16 = 2⁴), перевод между двоичной, восьмеричной и шестнадцатеричной системами можно делать мгновенно, минуя десятичную.

Триады (для 8-ричной системы)

Триада — это группа из трех битов. Каждая цифра 8-ричной системы идеально кодируется одной триадой.

  • 0₀ = 000, 7₈ = 111.
  • Чтобы перевести число 110101₂ в 8-ричную, бьем его на группы по 3 бита справа налево: 110 и 101. Получаем 65₈.

Тетрады (для 16-ричной системы)

Тетрада — это группа из четырех битов. Каждая цифра 16-ричной системы кодируется одной тетрадой.

  • 0₁₆ = 0000, F₁₆ (15) = 1111.
  • Переводим 10101111₂ в 16-ричную: бьем на группы по 4 бита: 1010 (это A) и 1111 (это F). Получаем AF₁₆.
📄 Коды чисел (Компьютерная арифметика)

Проблема вычитания в процессоре

Создавать отдельные электронные схемы для сложения и вычитания слишком дорого и сложно. Поэтому процессоры используют только сумматоры (схемы сложения). Вычитание реализуется как сложение с отрицательным числом: A - B = A + (-B). Для этого нужны специальные машинные коды.

1. Прямой код

Используется для хранения положительных чисел. Старший бит выделяется под знак (0 — плюс, 1 — минус).

Пример (8 бит): +5 = 00000101, -5 = 10000101.

2. Обратный код

Для положительных чисел совпадает с прямым. Для отрицательных: знаковый бит равен 1, а все значащие биты инвертируются (0 меняется на 1, 1 на 0).

Обратный для -5: 11111010.

3. Дополнительный код (Главный код ЭВМ)

Получается путем прибавления единицы (1) к младшему разряду обратного кода.

Дополнительный для -5: 11111010 + 1 = 11111011.

Зачем он нужен? При сложении положительного числа с его отрицательной версией в дополнительном коде, переполнение просто отбрасывается, и мы получаем ровно ноль аппаратно, без лишних проверок.

📄 Структурное программирование

Основы методологии

Структурное программирование — это парадигма, появившаяся в 1970-х годах для борьбы с "макаронным кодом" (spaghetti code), который получался из-за бесконтрольного использования оператора безусловного перехода goto.

Главные принципы:

  • Отказ от GOTO: использование только базовых конструкций (следование, ветвление, циклы).
  • Нисходящее проектирование (Top-Down): сложная задача разбивается на подзадачи (декомпозиция), пока они не станут тривиальными.
  • Модульность: выделение повторяющихся участков кода в подпрограммы (функции и процедуры). Каждый модуль должен быть независимым черным ящиком (иметь 1 вход и 1 выход).

Этот подход резко упростил чтение, тестирование и командную разработку сложных программ.

📄 Динамические структуры данных

В отличие от массивов с фиксированным размером, динамические структуры выделяют и освобождают память прямо во время выполнения программы по мере необходимости.

Списки

  • Односвязный список: набор узлов. Каждый узел хранит само значение и указатель (адрес) на следующий узел в памяти. Последний указывает на NULL.
  • Двусвязный список: узел хранит два указателя — на следующий и на предыдущий элемент. Можно двигаться в обе стороны.
Стек (Stack): принцип LIFO (Последним пришел — первым ушел). Аналогия: стопка книг. Операции: Push (добавить на вершину) и Pop (снять с вершины). Используется для рекурсии и отмены действий.
Очередь (Queue): принцип FIFO (Первым пришел — первым ушел). Аналогия: очередь в кассу. Операции: Enqueue (встать в конец) и Dequeue (выйти из начала). Используется для буферов и обработки задач.

Деревья

Иерархическая структура из узлов. Верхний — Корень. Узлы без детей — Листья. Бинарное дерево поиска (BST): у каждого узла максимум 2 потомка. Всё, что меньше родителя, идет влево, что больше — вправо. Обеспечивает очень быстрый поиск данных O(log N).

📄 Контейнеры и STL (C++)

Standard Template Library (STL)

Библиотека готовых структур данных (контейнеров) и алгоритмов в языке C++. Избавляет программиста от необходимости писать списки и деревья вручную.

Популярные контейнеры:

  • std::vector — Динамический массив. Память выделяется непрерывным куском. Плюсы: мгновенный доступ по индексу [i]. Минусы: медленная вставка или удаление в середине (приходится сдвигать все остальные элементы).
  • std::list — Двусвязный список. Плюсы: моментальная вставка и удаление в любом месте. Минусы: нельзя быстро обратиться к элементу по индексу (нужно перебирать с начала).
  • std::stack и std::queue — Адаптеры, реализующие стек и очередь.
  • std::map — Ассоциативный массив (словарь). Хранит данные парами «Ключ - Значение» (например, Имя - Телефон). Внутри реализовано на основе красно-черных деревьев. Ключи всегда автоматически сортируются.
  • std::set — Множество. Хранит только уникальные элементы. Дубликаты игнорируются при добавлении.
